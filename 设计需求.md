
既然我们的底层数据是一张统一的【时空编年总表】**，那在前端实现上，我们要做的就是一个**“多维画布的动态事件过滤器”**。为了让UI设计师（成员C）可以直接按照说明画图，前端开发（成员D）可以直接对照逻辑写控制函数，我将**整体网页布局**以及**三个核心场景做最终的工业级细化。

---

## 📐 一、 整体网页布局（Layout Blueprint）的像素级细化

整个网页是一个100%全屏、无滚动条（No-Scroll）的沉浸式仪表盘大屏，背景统一采用带微弱宣纸肌理的淡米色（`#F4F1EA`）。屏幕划分为以下四个精确的“瓦片区域”：

### 1. 【左半屏 50% 宽度】A区：笔墨山河·水墨数字地图

* **视觉呈现**：采用经过数字脱色、只保留山川脉络的古风中国地图（可利用 Mapbox 或 Leaflet 自定义瓦片图层）。
* **核心元素**：
* **标记点（Markers）**：地图上不使用现代图章，而是用微型**水墨晕染圆点**来代表地点（如杭州、南京、漓江、乐山）。
* **旅行线（Routes）**：当涉及多地连续游历时，点与点之间会由一条半透明、像毛笔擦笔划过的墨痕线串联。



### 2. 【右上四分之一屏 25% 宽 × 50% 高】B区：历史文献叙事窗

* **视觉呈现**：像一本竖排或横排的精致古籍卡片，常驻右上方。
* **核心元素**：
* **时间巨幕（Big Title）**：卡片顶部左上角，用巨大的书法体显示当前年份（如 `1949` 或 `己丑`），未选中时呈灰色，选中时变为焦墨黑。
* **滚动文献区（Scroll Container）**：带有自定义隐藏滚动条。展示当前行数据的“历史事件与文献记述”全文。文字行间距设定为 1.8 倍，确保博物馆级别的阅读体验。



### 3. 【右下四分之一屏 25% 宽 × 50% 高】C区：文人交游关系谱

* **视觉呈现**：基于 ECharts / Graph 或 D3.js force 的动态力导向图，背景色比网页主背景略暗，呈宣纸阴影色。
* **核心元素**：
* **人物气泡（Nodes）**：当前事件涉及的人物。气泡中心是名字，气泡边缘带有水墨毛边动效。
* **动态连线（Links）**：人与人之间的连线。



### 4. 【底部通栏 100% 宽 × 10% 高】D区：全景编年流水时间轴

* **视觉呈现**：横跨屏幕底部的复古刻度尺。
* **核心元素**：
* **时间刻度线**：每一个有数据的年份都是一个高亮刻度，中间没有数据的年份按比例压缩或留白。
* **悬停浮窗（Hover Tooltip）**：鼠标滑过刻度时，上方会浮现一个小气泡，显示“该年有 X 条历史记录”。

## 🔄 二、 三大场景的“深度扩展与代码逻辑”细化

为了让前端同学（成员D）能够轻松写出状态联动，我们需要把这三个场景翻译成“输入（触发条件）”**与**“输出（各区联动反应）”的详细规格。

### 🎬 场景一：时间轴驱动逻辑（以1949年己丑为例）

当用户点击底部时间轴（D区）的“1949”或“己丑”刻度时：

* **1. 数据过滤流（Backend Logic）：**
前端 JavaScript 触发监听函数，直接在总表数组中执行：
`let currentData = total_table.filter(item => item.year === 1949);`
系统提取出《慈竹平安馆图》相关的整段长文本。
* **2. 地图（A区）协同：**
* 地图接收到坐标指令，调用 `map.flyTo([110.29, 25.27], zoomLevel: 8)`，平滑漂移并放大聚焦到 **广西漓江与盖山** 区域。
* 该坐标点对应的水墨标志执行一个持续 1.5 秒的“墨汁扩散圈”动画。


* **3. 文献窗（B区）协同：**
* 右上方的大数字瞬间变为 `1949`，副标题变为 `己丑（八十六叟）`。
* 文字流载入黄宾虹的自述：“余游漓江，宿盖山对岸旬余……得图而归，写此一角……”


* **4. 关系谱（C区）协同：**
* 关系图重构，生成以【黄宾虹】和【雁迅】为核心的星空网络。
* 算法读取 1949 年文本中同时出现的【钟毓龙】、【田寄南】、【陈定谟】、【杨元生】。这四个人的气泡从四周向圆心靠拢，并与【黄宾虹】、【雁迅】连线。连线颜色从墨灰渐变为**朱砂红（`#C24F4F`）**。



---

### 🎬 场景二：人物节点驱动逻辑（以反向激活张宗祥为例）

当用户在右下角关系谱（C区）中，好奇点击了“张宗祥”这个气泡时：

* **1. 数据过滤流（Backend Logic）：**
系统触发反向检索：`let relatedRows = total_table.filter(item => item.figures.includes("张宗祥"));`
定位到涉及张宗祥的那一行（即 1950 年秋季题跋）。
* **2. 时间轴（D区）协同：**
* 底部时间轴自动产生平滑滚动，时间滑块（Slider）像磁铁一样被吸引，自动停留在 **【1950年 / 庚寅】** 刻度上，该刻度变为高亮状态。


* **3. 地图（A区）协同：**
* 地图视野从之前的广西漓江平滑切回 **浙江杭州**（因为这是张宗祥、钟毓龙等人的核心交游和题字地）。


* **4. 文献窗（B区）协同：**
* 右上方年份变更为 `1950`。文字刷新为张宗祥写的贺寿词：“竹比节，松祝寿。慈母恩，永不朽。”
* **【多媒体扩展操作】**：在文字右下方，触发一个 `CSS Keyframes` 动画：伴随着一声清脆的“下印”音效，一枚红色的“张宗祥印”虚拟印章在文本下方盖下，增强数字博物馆的“仪式感”。



---

### 🎬 场景三：地理空间驱动逻辑（以点击“巴蜀/四川”无年谱期为例）

当用户在左侧地图（A区）上，直接点击了巨大的“巴蜀/四川”区域标记时：

* **1. 数据过滤流（Backend Logic）：**
代码执行：`let subData = total_table.filter(item => item.location === "四川/巴蜀地区");`
此时筛选出后面关于“嘉定至夹江”、“大峨山石龛”、“瞿塘峡八阵图”等无明确公元纪年的多条数据。
* **2. 时间轴（D区）协同：**
* 底部普通的线性时间轴（1923-1950）整体变暗（Opacity: 0.3），代表退出严格的编年模式。
* 在时间轴最前方或上方，高亮弹出一个特殊的独立标签：**【游蜀行旅区间（时序考证中）】**。


* **3. 文献窗（B区）协同：**
* 整个右上方区域转变为“蜀游文史考证瀑布流”。
* 用户可以用鼠标向下滚动，页面会像翻阅考察日记一样，依次展现黄宾虹对巴蜀山川的记录（如：“大峨山在峨眉縣西南，有石龕一百十二……”、“堆碎石積成，出水數十丈……”）。


* **4. 关系谱（C区）协同：**
* 此时的关系谱不再是普通的“朋友圈”，而是演变为“历史互文网络”。
* 屏幕上亮起黄宾虹与【郭璞】、 【苏辙】、【陆游】、【诸葛亮】等古人的时空跨界连接（因为他在考察这些景点时，引证了这些古人的诗词名篇）。点击郭璞，就会高亮苏辙那句“有古郭璞生，此地苦箋注”的诗。

## 🛠️ 三、 终极项目任务书：给各组员的一键操作指南

为了让你的四人团队马上能动工，你现在可以把下面这段极度明确的任务描述，直接私发给你的 **设计师（成员C）** 和 **开发（成员D）**：

### 📋 给 @UI设计师（成员C） 的操作单

1. **画布尺寸**：请在 Figma 中建立一个 `1920x1080` 的单页画布，不要做多页面切换。
2. **布局划分**：左边 50% 留给地图；右边横向平分，上面放古籍卡片（文献），下面放社交星空（关系图）；最下面留 10% 做一条横向的长线（时间轴）。
3. **视觉素材准备**：
* 去找一张淡米色的宣纸背景图。
* 把成员A处理好的表格里提及的几枚印章（“宗祥诗书”、“临海徐氏”、“寄翁”）在 Photoshop 里抠成背景透明的红色 `.png` 图片，留给开发做盖章动效。



### 📋 给 @前端开发（成员D） 的操作单

1. **技术栈选型**：推荐直接使用 `Vue.js / React` 作为主框架，图表部分 100% 推荐使用 `Apache ECharts`（因为它同时自带 `geo` 地图组件和 `graph` 力导向关系图组件，最容易做联动）。
2. **数据灌入**：把成员A给你的这门【时空编年总表】CSV 文件，在线转换成一个标准的 JSON 数组，作为全局变量 `const binHongData = [...]`。
3. **核心联动函数（伪代码参考）**：
```javascript
// 建立一个中央控制函数，任何人操作都调用它
function updateGlobalState(selectedRow) {
    Timeline.activeYear(selectedRow.year);      // 1. 底部时间轴跳动
    MapBox.panTo(selectedRow.coordinates);      // 2. 左侧地图飞过去
    DocWindow.renderText(selectedRow.content);  // 3. 右上角文字刷新并盖章
    RelationGraph.refresh(selectedRow.figures); // 4. 右下角关系网重组
}

```
